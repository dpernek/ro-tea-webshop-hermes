## 1. Sažetak izdanja

Ovo izdanje predstavlja stabilnu verziju RO-TEA webshopa nakon više faza razvoja, popravaka i stabilizacije. Webshop je operativan: kupci mogu pregledavati proizvode, dodavati u košaricu, završavati narudžbe (karticom, bankovnom uplatom i pouzećem), a admin može upravljati proizvodima, narudžbama, GLS pošiljkama i sadržajem početne stranice.

Sustav je prošao fazu završnog QA-a. Nisu pronađeni P0/P1 bugovi.

## 3. Ključne poslovne mogućnosti

### Što kupac može
- Pregledavati katalog profesionalnih alata i opreme (847 proizvoda)
- Filtrirati po kategoriji, brendu, pretraživati
- Vidjeti cijenu (redovnu i akcijsku), stanje zalihe, EAN, opis, specifikacije
- Dodati proizvod u košaricu, primijeniti kupon

### Što admin može

- Pregledavati i obrađivati narudžbe (promjena statusa, plaćanja, interna napomena)
- Kreirati GLS pošiljke (home delivery i Paketomat), stornirati ih
- Uređivati sadržaj početne stranice (hero, trust sekcija, CTA)
- Upravljati kuponima, dostavom, PDF katalozima, korisnicima
- Pregledavati audit log aktivnosti
- Izvesti narudžbe u CSV

## 4. Poznate otvorene stavke / backlog

### P2 / UX poboljšanja
- "Pouzeće" quick chip na `/admin/orders` ne filtrira po `paymentMethod` (nedostaje filter u API-ju)
- Admin order detail activity block prikazuje samo ručne audit akcije, ne automatske promjene statusa
- Product cards: kategorija prikazana kao string, ne link

### Tehnički dug
- 27 lint warnings (pre-existing, nisu kritični)
- Neiskorišteni importi (ForbiddenCard, Loader2, Plus, Search) u nekoliko admin pageva
- Dupli shipping lookup u Stripe checkout ruti (line 100–101) — kozmetički
- `force-dynamic` na 60 admin API ruta — dio su nužni, dio povijesni

### Operativna poboljšanja
- Inventory management: nema low-stock prag definiran u settingsu (trenutno hardcodiran ≤3)
- GLS status refresh ne radi za Paketomat dok GLS ne dodijeli broj za praćenje — svjestan fallback
- Nema automatske notifikacije za nove narudžbe (samo email)

### Future enhancements
- Dashboard: revenue po periodima (7 dana, 30 dana) umjesto samo lifetime
- Orders export: dodati shippingMethod, couponCode u CSV
- Batch operacije nad narudžbama (mark viewed, bulk status)
- GLS widget: map picker na checkoutu već postoji, UI dorada moguća
- Product SEO: meta description iz product detail podataka
- Image optimization: WebP konverzija pri uploadu

## 5. Rizici / napomene za budući rad

### Osjetljiva područja
- **Prisma schema izmjene**: svaka izmjena sheme zahtijeva `prisma generate` i može uzrokovati schema-drift. Svi javni queryji sada imaju eksplicitni `select` — **nove queryje treba raditi s `select`, nikad s `include: true`**.
- **GLS REST payload**: GLS API contract je točno mapiran. Svaka izmjena payloada (ServiceList, PSDParameter) treba testirati kroz stvarni GLS create.
- **Checkout pricing**: Server-side `computePrices` koristi `salePrice`. Ne mijenjati pricing logiku bez testiranja ukupnog totala.
- **Rate limiting**: `proxy.ts` sadrži login i admin write limiting. In-memory store ne dijeli state između Vercel edge nodeova.

### Deploy gate
- Obavezno: `npm run lint` + `npm run build` prije pusha
- Obavezno: `node scripts/smoke-production.mjs` nakon deploya
- Vercel rate limit: 100 deploys/dan — planirati deployeve
